## Objetivo

Implementar o conjunto de páginas do grupo `(dashboard)` que permitem ao professor fazer login, visualizar estatísticas, gerenciar quizzes (criar, editar, publicar, deletar) e editar perguntas com reordenação via drag-and-drop.

**Por quê:** É o núcleo do fluxo do professor — sem estas páginas não há como criar conteúdo para as sessões de jogo.

## Regras de Negócio

1. Só o autor do quiz pode editá-lo, deletá-lo ou publicá-lo (validação no backend; frontend não exibe ações de quizzes de outros autores pois `GET /api/quizzes` já filtra por autor)
2. Um quiz só pode ser publicado se tiver pelo menos 1 pergunta — botão "Publicar" fica disabled e com tooltip explicativo
3. Cada pergunta deve ter entre 2 e 4 opções de resposta (validação visual; API não impõe esse limite)
4. Exatamente 1 opção por pergunta deve ser `isCorrect: true` — ao marcar uma opção como correta, as demais são desmarcadas automaticamente
5. Ao deletar um quiz, modal de confirmação é exibido antes de chamar a API
6. A reordenação de perguntas é **otimista**: UI atualiza imediatamente, API confirma em seguida; em caso de falha, reverte para ordem original com mensagem de erro
7. Um quiz publicado pode ser colocado de volta como rascunho (`isPublished: false`) com a mesma ação de toggle

## Critérios de Aceite

### Login (`/login`)
- **Dado** que o professor acessa `/login`
- **Quando** preenche email e senha corretos e submete
- **Então** é redirecionado para `/dashboard`

- **Dado** que o professor preenche credenciais inválidas
- **Quando** submete
- **Então** vê mensagem de erro "Credenciais inválidas" abaixo do formulário

- **Dado** que o professor clica em submeter
- **Quando** a requisição está em andamento
- **Então** o botão exibe spinner e fica desabilitado

### Dashboard (`/dashboard`)
- **Dado** que o professor está autenticado em `/dashboard`
- **Quando** a página carrega
- **Então** vê: total de quizzes, quantidade de publicados, quantidade de rascunhos e link para criar novo quiz

### Lista de Quizzes (`/quizzes`)
- **Dado** que o professor está em `/quizzes`
- **Quando** a lista carrega
- **Então** vê cada quiz com: título, badge de status (Publicado/Rascunho), nº de perguntas e ações

- **Dado** que o professor clica em "Deletar"
- **Quando** confirma no diálogo de confirmação
- **Então** o quiz é removido da lista

- **Dado** que a lista está vazia
- **Quando** a página carrega
- **Então** vê empty state com call-to-action "Criar seu primeiro quiz"

- **Dado** que o professor clica em "Publicar" num quiz com 0 perguntas
- **Quando** a ação é executada
- **Então** vê erro "O quiz precisa ter pelo menos 1 pergunta para ser publicado"

### Criar Quiz (`/quizzes/create`)
- **Dado** que o professor preenche título e submete
- **Quando** a API confirma a criação
- **Então** é redirecionado para `/quizzes/{id}` (editor de perguntas)

- **Dado** que o campo título está vazio
- **Quando** o professor tenta submeter
- **Então** vê erro de validação "Título obrigatório" inline

### Editor de Perguntas (`/quizzes/[id]`)
- **Dado** que o professor clica em "Adicionar Pergunta"
- **Quando** preenche o formulário e salva
- **Então** a nova pergunta aparece na lista

- **Dado** que o professor arrasta uma pergunta para nova posição
- **Quando** solta
- **Então** a lista atualiza visualmente de imediato e a nova ordem é persistida via API

- **Dado** que o professor adiciona uma opção de resposta com texto e cor
- **Quando** salva
- **Então** a opção aparece na pergunta com a cor visual correspondente

- **Dado** que o professor marca uma opção como correta
- **Quando** outra opção já estava marcada como correta
- **Então** a marcação anterior é removida automaticamente

## Edge Cases

1. Quiz com 0 perguntas → botão "Publicar" disabled com tooltip "Adicione ao menos 1 pergunta para publicar"
2. Erro de rede ao salvar pergunta → toast de erro + dados do formulário preservados
3. Sessão expirada durante edição → middleware do Next.js redireciona para `/login` (automático)
4. Lista de quizzes vazia → empty state com CTA "Criar seu primeiro quiz"
5. Reorder falha na API → UI reverte para ordem original com mensagem de erro via toast
6. Drag-and-drop com uma única pergunta → lista exibe normalmente, sem drag handle visível
7. Duas opções marcadas como corretas → ao marcar a segunda, a primeira é desmarcada automaticamente
8. Título com 200 caracteres → aceito; 201 → erro de validação inline no formulário
9. Professor tenta acessar `/quizzes` sem sessão → middleware redireciona para `/login`
10. Formulário de pergunta com `timeLimitSecs` < 5 → erro de validação "Mínimo 5 segundos"

- Sidebar com `<Link>` para `/dashboard` e `/quizzes`
- Header com `session.user.name` e botão de logout (`signOut`)

### `/login` — Página de Login

- Logo + título "Azimute"
- Card centralizado com: label Email, `input-user-email`, label Senha, `input-user-password`
- `btn-submit-login` → "Entrar"
- Área de erro inline (`data-testid="text-login-error"`)
- Estados: idle / loading (spinner no botão + disabled) / error

### `/dashboard` — Visão Geral

- 3 cards de estatísticas: `card-total-quizzes`, `card-published-quizzes`, `card-draft-quizzes`
- `btn-create-quiz` → "Criar Novo Quiz"
- Lista dos últimos 5 quizzes com título e badge de status

### `/quizzes` — Lista de Quizzes

- `btn-create-quiz` no topo direito
- Grid/tabela com por item: título, `badge-quiz-status-{id}`, contador de perguntas, data
- Ações por item: `btn-edit-quiz-{id}`, `btn-publish-quiz-{id}`, `btn-delete-quiz-{id}`
- `modal-confirm-delete` com confirmação antes de deletar
- Estado loading: skeleton rows
- Estado empty: mensagem + CTA

### `/quizzes/create` — Criar Quiz

- `input-quiz-title` (obrigatório)
- `input-quiz-description` (opcional)
- `btn-submit-create-quiz` → "Criar Quiz"
- Erro de validação inline por campo

### `/quizzes/[id]` — Editor de Perguntas

- Header: título do quiz + `badge-quiz-status` + `btn-publish-quiz` + `btn-back-to-quizzes`
- Lista sortable de perguntas via @dnd-kit (drag handle por item)
- Por pergunta:
  - Texto, tempo (segundos), pontos
  - Lista de opções com cor visual (vermelho/azul/amarelo/verde)
  - `btn-delete-question-{id}`, `btn-add-option-{questionId}`
- `btn-add-question` no final da lista
- `QuestionForm` inline (acordeão) para nova pergunta
- `AnswerOptionForm` inline para nova opção

### Contratos de API consumidos

| Endpoint | Método | Usado em |
|---|---|---|
| `POST /auth/login` | POST | `useLogin` (via NextAuth `signIn`) |
| `GET /api/quizzes` | GET | `useQuizzes` |
| `GET /api/quizzes/:id` | GET | `useQuizDetail` |
| `POST /api/quizzes` | POST | `useQuizActions.createQuiz` |
| `PATCH /api/quizzes/:id` | PATCH | `useQuizActions.updateQuiz` |
| `DELETE /api/quizzes/:id` | DELETE | `useQuizActions.deleteQuiz` |
| `PATCH /api/quizzes/:id/publish` | PATCH | `useQuizActions.publishQuiz` |
| `POST /api/quizzes/:quizId/questions` | POST | `useQuizEditor.addQuestion` |
| `PATCH /api/questions/:id` | PATCH | `useQuizEditor.updateQuestion` |
| `DELETE /api/questions/:id` | DELETE | `useQuizEditor.deleteQuestion` |
| `PATCH /api/questions/:id/reorder` | PATCH | `useQuizEditor.reorderQuestion` |
| `POST /api/questions/:questionId/options` | POST | `useQuizEditor.addOption` |
| `PATCH /api/options/:id` | PATCH | `useQuizEditor.updateOption` |
| `DELETE /api/options/:id` | DELETE | `useQuizEditor.deleteOption` |

### Padrão TanStack Query

```typescript
// Query Keys (centralizados em features/quiz/hooks/query-keys.ts)
export const quizKeys = {
  all: ['quizzes'] as const,
  lists: () => [...quizKeys.all, 'list'] as const,
  detail: (id: string) => [...quizKeys.all, 'detail', id] as const,
}

// Após mutation bem-sucedida: invalidar query relevante
queryClient.invalidateQueries({ queryKey: quizKeys.lists() })
queryClient.invalidateQueries({ queryKey: quizKeys.detail(quizId) })
```
