## Objetivo

Implementar endpoints REST para criar, editar, reordenar e remover `Question` e `AnswerOption` de quizzes no Azimute. Esses dados são a base do engine do jogo — cada pergunta precisa ter exatamente 1 opção correta e tempo de resposta válido antes de entrar em uma partida.

---

## Modelo de Dados

O schema Prisma já contém `Question` e `AnswerOption` com `onDelete: Cascade`. **Nenhuma migração é necessária.**

### Question (já no schema)

```prisma
model Question {
  id            String         @id @default(cuid())
  quiz          Quiz           @relation(fields: [quizId], references: [id], onDelete: Cascade)
  quizId        String
  text          String
  mediaType     MediaType?
  mediaUrl      String?
  timeLimitSecs Int            @default(30)
  points        Int            @default(1000)
  order         Int
  options       AnswerOption[]
  gameAnswers   GameAnswer[]

  @@index([quizId, order])
}
```

### AnswerOption (já no schema)

```prisma
model AnswerOption {
  id         String      @id @default(cuid())

}
```

## Regras de Negócio

1. **Autenticação obrigatória** — todas as rotas exigem `requireAuth`
2. **Autorização de propriedade** — apenas o autor do quiz (`quiz.authorId === user.id`) ou ADMIN pode criar/editar/deletar perguntas e opções
3. **timeLimitSecs** — entre 5 e 120 segundos (inclusive)
4. **Máximo de opções** — 4 por pergunta; tentativa de criar a 5ª retorna 422
5. **Opção correta única** — ao criar ou atualizar uma opção com `isCorrect: true`, resetar todas as outras opções da mesma pergunta para `isCorrect: false` em transação atômica (`prisma.$transaction`)
6. **order automático** — se não fornecido na criação, calcular como `MAX(order) + 1` entre os registros existentes (1 se for o primeiro)
7. **CASCADE garantido pelo schema** — `DELETE Question` remove todas as `AnswerOption` filhas sem lógica adicional na aplicação

## Critérios de Aceite

### POST /api/quizzes/:quizId/questions

**Dado** que um TEACHER autenticado é autor do quiz `quizId`  
**Quando** envia `POST` com body válido  
**Então** retorna `201` com `QuestionPublic` (options: []) e `order` calculado

**Dado** que o quiz não pertence ao TEACHER autenticado  
**Quando** envia `POST`  
**Então** retorna `403`

**Dado** que o quiz não existe  
**Quando** envia `POST`  
**Então** retorna `404`

**Dado** que `timeLimitSecs` < 5 ou > 120  
**Quando** envia `POST`  
**Então** retorna `400`

---

### PATCH /api/questions/:id

**Dado** que o TEACHER é autor do quiz que contém a pergunta  
**Quando** envia campos parciais válidos  
**Então** retorna `200` com `QuestionPublic` atualizado

**Dado** que o body é vazio (`{}`)  
**Quando** envia `PATCH`  
**Então** retorna `400`

**Dado** que a pergunta não existe  
**Quando** envia `PATCH`  
**Então** retorna `404`

**Dado** que o TEACHER não é autor do quiz  
**Quando** envia `PATCH`  
**Então** retorna `403`

---

### DELETE /api/questions/:id

**Dado** que o TEACHER é autor do quiz  
**Quando** envia `DELETE`  
**Então** retorna `204` e todas as opções filhas são removidas (cascade)

**Dado** que o TEACHER não é autor do quiz  
**Quando** envia `DELETE`  
**Então** retorna `403`

**Dado** que a pergunta não existe  
**Quando** envia `DELETE`  
**Então** retorna `404`

---

### PATCH /api/questions/:id/reorder

**Dado** que o TEACHER é autor do quiz  
**Quando** envia `{ order: 2 }`  
**Então** retorna `200` com apenas o campo `order` atualizado

---

### POST /api/questions/:questionId/options

**Dado** que o TEACHER é autor do quiz e a pergunta tem < 4 opções  
**Quando** envia body válido com `isCorrect: false`  
**Então** retorna `201` com `AnswerOptionPublic`

**Dado** que body tem `isCorrect: true`  
**Quando** envia `POST`  
**Então** retorna `201` e todas as outras opções da pergunta têm `isCorrect: false`

**Dado** que a pergunta já tem 4 opções  
**Quando** envia `POST`  
**Então** retorna `422` com mensagem `"Máximo de 4 opções por pergunta atingido"`

**Dado** que o TEACHER não é autor do quiz  
**Quando** envia `POST`  
**Então** retorna `403`

---

### PATCH /api/options/:id

**Dado** que o TEACHER é autor do quiz e envia `{ isCorrect: true }`  
**Quando** envia `PATCH`  
**Então** retorna `200` e as outras opções da mesma pergunta ficam com `isCorrect: false`

**Dado** que o body é vazio  
**Quando** envia `PATCH`  
**Então** retorna `400`

**Dado** que a opção não existe  
**Quando** envia `PATCH`  
**Então** retorna `404`

---

### DELETE /api/options/:id

**Dado** que o TEACHER é autor do quiz  
**Quando** envia `DELETE`  
**Então** retorna `204`

**Dado** que o TEACHER não é autor do quiz  
**Quando** envia `DELETE`  
**Então** retorna `403`

**Dado** que a opção não existe  
**Quando** envia `DELETE`  
**Então** retorna `404`

## Edge Cases

1. **Quiz sem perguntas** — `order` calculado como `1` na primeira criação
2. **Body vazio em PATCH** — retornar `400` (Zod `.refine(data => Object.keys(data).length > 0, ...)`)
3. **color inválida** — retornar `400` (Zod enum `['RED','BLUE','YELLOW','GREEN']`)
4. **timeLimitSecs = 5** — válido (boundary mínimo)
5. **timeLimitSecs = 120** — válido (boundary máximo)
6. **timeLimitSecs = 4 ou 121** — inválido, retornar `400`
7. **5ª opção** — retornar `422` com mensagem clara (novo `UnprocessableEntityError`)
8. **ADMIN** — pode editar/deletar perguntas e opções de qualquer quiz (`role === 'ADMIN'` bypassa verificação de `authorId`)
9. **DELETE opção única isCorrect** — permitido no MVP; a validação de "exatamente 1 correta" é responsabilidade do CRUD de Quizzes (publicação)
10. **mediaUrl com mediaType ausente** — `400`; inversamente, `mediaType` sem `mediaUrl` também `400` (validação cruzada com `refine` no schema)

## Arquitetura

### Decisão de design

Dois módulos separados (`question/` e `option/`). O `option.service.ts` chama o Prisma para obter a relação `option → question → quiz` e verificar ownership. A lógica de validação cruzada (max 4 opções, reset isCorrect) vive em `option.service.ts`.

### question.service.ts — interface pública

```typescript
createQuestion(quizId: string, body: CreateQuestionBody, user: AuthenticatedUser): Promise<QuestionPublic>
updateQuestion(id: string, body: UpdateQuestionBody, user: AuthenticatedUser): Promise<QuestionPublic>
deleteQuestion(id: string, user: AuthenticatedUser): Promise<void>
reorderQuestion(id: string, order: number, user: AuthenticatedUser): Promise<QuestionPublic>
```

Lógica interna:
- `assertQuizOwnership(quizId, user)` — helper privado que lança `NotFoundError` ou `ForbiddenError`
- `getNextOrder(quizId)` — `MAX(order) + 1` ou `1` se vazio

### option.service.ts — interface pública

```typescript
createOption(questionId: string, body: CreateOptionBody, user: AuthenticatedUser): Promise<AnswerOptionPublic>
updateOption(id: string, body: UpdateOptionBody, user: AuthenticatedUser): Promise<AnswerOptionPublic>
deleteOption(id: string, user: AuthenticatedUser): Promise<void>
```

Lógica interna:
- `assertOptionOwnership(optionId, user)` — segue a cadeia `option → question → quiz → authorId`
- `assertQuestionOptionOwnership(questionId, user)` — segue `question → quiz → authorId`
- Ao `isCorrect: true` em create ou update: `prisma.$transaction` que reseta as outras opções e salva a nova/atualizada

### Schemas Zod

**question.schemas.ts:**

```typescript
export const CreateQuestionBodySchema = z.object({
  text: z.string().min(1).max(500),
  mediaType: z.enum(['IMAGE', 'VIDEO']).optional(),
  mediaUrl: z.string().url().optional(),
  timeLimitSecs: z.number().int().min(5).max(120).default(30),
  points: z.number().int().min(100).max(2000).default(1000),
  order: z.number().int().positive().optional(),
}).refine(
  (d) => !(d.mediaUrl && !d.mediaType) && !(d.mediaType && !d.mediaUrl),
  { message: 'mediaType e mediaUrl devem ser fornecidos juntos' }
)

export const UpdateQuestionBodySchema = z.object({
  text: z.string().min(1).max(500).optional(),
  mediaType: z.enum(['IMAGE', 'VIDEO']).nullable().optional(),
  mediaUrl: z.string().url().nullable().optional(),
  timeLimitSecs: z.number().int().min(5).max(120).optional(),
  points: z.number().int().min(100).max(2000).optional(),
  order: z.number().int().positive().optional(),
}).refine((d) => Object.keys(d).length > 0, { message: 'Ao menos um campo deve ser fornecido' })

export const ReorderQuestionBodySchema = z.object({
  order: z.number().int().positive(),
})
```
