# Spec: Backend — Autenticação de Professores (Auth.js v5)

**Ticket:** `planka-1773039018364109944`
**Branch:** `feat/planka-1773039018364109944-auth-professores`
**Criado em:** 2026-05-12

---

## Objetivo

Implementar autenticação segura para professores no sistema Azimute. O professor precisa fazer login com email/senha para acessar o dashboard. A autenticação usa Auth.js v5 (NextAuth) no Next.js como gerenciador de sessão e o Express API como validador de credenciais.

**Por que essa separação:** O Express API é a fonte da verdade para dados; o NextAuth gerencia o ciclo de vida da sessão (JWT stateless, sem necessidade de Redis para sessão).

---

## Modelo de Dados

Já existe no Prisma schema — **nenhuma migração necessária**.

```prisma
model User {
  id           String   @id @default(cuid())
  name         String
  email        String   @unique
  passwordHash String           // Argon2id, time=3, mem=64MB
  role         Role     @default(TEACHER)
  createdAt    DateTime @default(now())
  updatedAt    DateTime @updatedAt
}

enum Role {
  TEACHER
  ADMIN
}
```

**Tipos TypeScript estendidos (next-auth.d.ts):**

```ts
// Extensão necessária para adicionar `id` e `role` à sessão NextAuth
interface Session {
  user: { id: string; role: 'TEACHER' | 'ADMIN' } & DefaultSession['user']
}
interface User { role: 'TEACHER' | 'ADMIN' }
interface JWT  { id: string; role: 'TEACHER' | 'ADMIN' }
```

## Regras de Negócio

1. **Registro (primeiro admin):** `POST /auth/register` só funciona se não houver nenhum usuário no banco. Se já houver ≥ 1 usuário → 409 Conflict.
2. **Hash de senha:** Argon2id com `timeCost: 3`, `memoryCost: 65536` (64 MB). A lib `argon2` já está instalada no API.
3. **Login:** Buscar usuário por email. Verificar hash com `argon2.verify`. Se falhar (email não encontrado ou senha errada) → 401 com mensagem genérica `"Credenciais inválidas"` (não revelar qual campo está errado — prevenção de enumeração de emails).
4. **Sessão:** JWT stateless gerenciado pelo NextAuth. Expiração configurável via `AUTH_TRUST_HOST` + `session.maxAge`.
5. **CORS:** O Express API já tem CORS configurado para aceitar apenas a origem do Next.js via `process.env['NEXTAUTH_URL']` — nenhuma mudança necessária.
6. **Helmet:** Já configurado no `createApp()` — nenhuma mudança necessária.

## Critérios de Aceite

```
Dado que não há usuários no banco
Quando POST /auth/register com { name, email, password válidos }
Então retorna 201 com { id, name, email, role: "ADMIN", createdAt }
  E o usuário é persistido com passwordHash Argon2id

Dado que já há ≥ 1 usuário no banco
Quando POST /auth/register
Então retorna 409 com { error: "Registro disponível apenas para o primeiro administrador" }

Dado que o professor tem conta no banco
Quando POST /auth/login com { email, password } corretos
Então retorna 200 com { id, name, email, role }

Dado que as credenciais são inválidas (email errado ou senha errada)
Quando POST /auth/login
Então retorna 401 com { error: "Credenciais inválidas" }

Dado que o professor chamou POST /auth/login com sucesso
Quando o NextAuth recebe os dados do usuário em authorize()
Então cria sessão JWT com { id, name, email, role } e redireciona para /dashboard

Dado que a sessão JWT é válida
Quando o professor acessa /dashboard (ou sub-rotas)
Então o middleware permite o acesso

Dado que não há sessão JWT ou ela expirou
Quando o usuário tenta acessar /dashboard
Então o middleware redireciona para /login

Dado que o professor está autenticado
Quando chama signOut()
Então a sessão é destruída e é redirecionado para /login
```

## Edge Cases

1. **Registro concorrente:** Duas requisições simultâneas de `/auth/register` com banco vazio → o constraint `@unique` do Prisma garante que apenas um sucede; o outro recebe 409 (tratado como "já há usuário").
2. **Password muito curta:** < 8 chars → 400 com mensagem de validação Zod.
3. **Email malformado:** Email inválido no body → 400.
4. **Body vazio ou malformado:** JSON inválido → 400 (tratado pelo `express.json()`).
5. **Rate limiting:** O rate limit global (100 req / 15 min) cobre a rota de login; brute-force com IPs individuais é atenuado.
6. **Sessão expirada durante navegação:** NextAuth redireciona automaticamente para `/login` via middleware.
7. **`NEXT_PUBLIC_API_URL` não configurado:** O `authorize()` tenta fetch para `undefined` → retorna `null` (autenticação falha com erro genérico). Documentado como pré-requisito.

## Arquitetura

### Fluxo completo

```
Professor (browser)
  → POST /api/auth/signin  (NextAuth route handler)
      → authorize()
          → POST http://localhost:4000/auth/login  (Express API)
              → prisma.user.findUnique({ where: { email } })
              → argon2.verify(passwordHash, password)
              → retorna { id, name, email, role }
      → jwt() callback: adiciona id e role ao token
      → session() callback: expõe id e role na sessão
  → sessão JWT salva em cookie httpOnly

Acesso ao dashboard:
  → middleware.ts intercepta /dashboard/*
  → auth() verifica JWT do cookie
  → sem sessão → redirect /login
  → com sessão → permite request
```

### Implementação — API (Express)

**`auth.schema.ts`**
```ts
import { z } from 'zod'

export const RegisterBodySchema = z.object({
  name: z.string().min(1).max(100),
  email: z.string().email(),
  password: z.string().min(8).max(128),
})

export const LoginBodySchema = z.object({
  email: z.string().email(),
  password: z.string().min(1),
})

export type RegisterBody = z.infer<typeof RegisterBodySchema>
export type LoginBody = z.infer<typeof LoginBodySchema>
```

**`auth.service.ts`**
```ts
import argon2 from 'argon2'
import { prisma } from '../../config/database.js'
import { ConflictError, UnauthorizedError } from '../../shared/errors/http-errors.js'

const ARGON2_OPTIONS = { type: argon2.argon2id, timeCost: 3, memoryCost: 65536 } as const

export async function createFirstAdmin(name: string, email: string, password: string) {
  const count = await prisma.user.count()
  if (count > 0) throw new ConflictError('Registro disponível apenas para o primeiro administrador')
  const passwordHash = await argon2.hash(password, ARGON2_OPTIONS)
  return prisma.user.create({ data: { name, email, passwordHash, role: 'ADMIN' },
    select: { id: true, name: true, email: true, role: true, createdAt: true } })
}

export async function validateCredentials(email: string, password: string) {
  const user = await prisma.user.findUnique({ where: { email } })
  if (!user) throw new UnauthorizedError('Credenciais inválidas')
  const valid = await argon2.verify(user.passwordHash, password)
  if (!valid) throw new UnauthorizedError('Credenciais inválidas')
  return { id: user.id, name: user.name, email: user.email, role: user.role }
}
```

**`auth.router.ts`**
```ts
import { Router } from 'express'
import { LoginBodySchema, RegisterBodySchema } from './auth.schema.js'
import { createFirstAdmin, validateCredentials } from './auth.service.js'

export const authRouter = Router()

authRouter.post('/register', async (req, res, next) => {
  try {
    const body = RegisterBodySchema.parse(req.body)
    const user = await createFirstAdmin(body.name, body.email, body.password)
    res.status(201).json(user)
  } catch (err) { next(err) }
})

authRouter.post('/login', async (req, res, next) => {
  try {
    const body = LoginBodySchema.parse(req.body)
    const user = await validateCredentials(body.email, body.password)
    res.json(user)
  } catch (err) { next(err) }
})
```

### Implementação — Web (Next.js)

**`src/lib/auth.ts`** (modificado)
```ts
import NextAuth from 'next-auth'
import Credentials from 'next-auth/providers/credentials'

export const { handlers, auth, signIn, signOut } = NextAuth({
  providers: [
    Credentials({
      credentials: {
        email: { label: 'Email', type: 'email' },
        password: { label: 'Senha', type: 'password' },
      },
      async authorize(credentials) {
        const res = await fetch(`${process.env.NEXT_PUBLIC_API_URL}/auth/login`, {
          method: 'POST',
          body: JSON.stringify(credentials),
          headers: { 'Content-Type': 'application/json' },
        })
        if (!res.ok) return null
        return res.json() as Promise<{ id: string; name: string; email: string; role: string }>
      },
    }),
  ],
  callbacks: {
    jwt({ token, user }) {
      if (user) { token.id = user.id; token.role = (user as { role: string }).role }
      return token
    },
    session({ session, token }) {
      session.user.id = token.id as string
      session.user.role = token.role as string
      return session
    },
  },
  pages: { signIn: '/login' },
})
```

**`src/middleware.ts`** (novo)
```ts
import { auth } from '@/lib/auth'
import { ROUTES } from '@/constants/routes'

export default auth((req) => {
  if (!req.auth) {
    return Response.redirect(new URL(ROUTES.LOGIN, req.url))
  }
})

export const config = {
  matcher: ['/dashboard/:path*', '/quizzes/:path*'],
}
```
