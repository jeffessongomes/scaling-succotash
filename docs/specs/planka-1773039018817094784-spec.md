## Objetivo

Implementar o core da partida em tempo real: gerenciamento de sessões de jogo via Socket.io + Redis, com state machine explícita para transições de estado, timer automático no servidor, e REST endpoints para criação/consulta de sessões.

Este módulo é o **bloqueador do MVP** — conecta professores (host), alunos (players) e tela de projeção (display) durante uma partida ao vivo.

## State Machine

### Diagrama de Transições

```
LOBBY
  └─► ACTIVE        (game:start)
        └─► QUESTION    (game:next-question — automático ao entrar em ACTIVE)
              └─► REVEAL     (game:reveal-answers | timer expirado)
                    └─► LEADERBOARD  (game:show-leaderboard)
                          ├─► QUESTION    (game:next-question — mais perguntas)
                          └─► FINISHED   (game:end — última pergunta)

ANY STATUS ──► FINISHED    (game:end — abort)
```

### Implementação (`game.state-machine.ts`)

```typescript
const VALID_TRANSITIONS: Record<SessionStatus, SessionStatus[]> = {
  LOBBY:       ['ACTIVE', 'FINISHED'],
  ACTIVE:      ['QUESTION', 'FINISHED'],
  QUESTION:    ['REVEAL', 'FINISHED'],
  REVEAL:      ['LEADERBOARD', 'FINISHED'],
  LEADERBOARD: ['QUESTION', 'FINISHED'],
  FINISHED:    [],
}

function canTransition(from: SessionStatus, to: SessionStatus): boolean {
  return VALID_TRANSITIONS[from].includes(to)
}

function assertTransition(from: SessionStatus, to: SessionStatus): void {
  if (!canTransition(from, to)) {
    throw new InvalidTransitionError(from, to)
  }
}
```

## Regras de Negócio

1. **PIN único de 6 dígitos**: gerado aleatoriamente, verificado no Redis antes de salvar (até 10 tentativas).
2. **Quiz publicado**: sessão só pode ser criada com `quiz.isPublished = true`.
3. **Sessão ativa única**: um quiz não pode ter mais de uma sessão não-`FINISHED` ativa simultaneamente.
4. **Autenticação de host**: JWT Bearer obrigatório na criação REST. No socket, o host autentica via `auth.token` no handshake.
5. **Jogadores**: entram pelo PIN + nickname único por sessão. `avatarId` deve ser `"1"` a `"10"`.
6. **Controle exclusivo do host**: eventos de controle (`game:start`, `game:next-question`, etc.) só são aceitos do socket identificado como `hostSocketId` no estado da sessão.
7. **Pontuação por tempo**:
   ```
   rawPoints = points_base × (timeLimitMs − answeredInMs) / timeLimitMs
   finalPoints = Math.max(rawPoints, points_base × 0.5)
   ```
   Sem resposta ou fora do tempo: `0` pontos.
8. **Timer automático**: ao entrar em `QUESTION`, o servidor agenda `setTimeout(timeLimitSecs × 1000)` para acionar `game:reveal-answers` automaticamente. Se o host revelar antes, o timer é cancelado.
9. **Desconexão do host**: aguardar 30s via `setTimeout`. Se o host não reconectar, emitir `session:error` (`HOST_DISCONNECTED`) para toda a sala e finalizar a sessão.
10. **Desconexão de jogador**: manter na sessão com `isConnected: false`. Reconexão pelo mesmo nickname/PIN restaura o estado.
11. **Cleanup ao FINISHED**: persistir `GameAnswer` e scores no PostgreSQL, deletar chaves Redis, emitir `session:ended`.
12. **Rate limiting de socket**: máx. `env.GAME_WS_RATE_LIMIT` (10) eventos/socket/segundo.

## Critérios de Aceite

**Criar sessão (REST):**
- Dado um professor autenticado com quiz publicado
- Quando `POST /api/sessions { quizId }`
- Então `201` com `{ pin, sessionId, status: "LOBBY" }`, PIN salvo no Redis e no PostgreSQL

**Jogador entra no lobby:**
- Dado PIN válido, nickname único e avatarId entre "1" e "10"
- Quando socket emite `player:join { pin, nickname, avatarId }`
- Então `session:player-joined` emitido para toda a sala com `{ participant, totalPlayers }`

**Iniciar partida:**
- Dado host conectado com sessão em `LOBBY` com pelo menos 1 jogador
- Quando host emite `game:start { pin }`
- Então status → `ACTIVE` → `QUESTION`, `session:question-start` emitido (sem `isCorrect` nas opções), timer agendado no servidor

**Jogador responde:**
- Dado sessão em `QUESTION` e `answeredInMs <= timeLimitMs`
- Quando socket emite `player:answer { pin, questionId, optionId, answeredInMs }`
- Então pontuação calculada e salva no Redis Hash, `session:answer-received { answeredCount, totalPlayers }` emitido apenas ao host

**Timer expira:**
- Dado sessão em `QUESTION` e timer do servidor disparado
- Então transição automática para `REVEAL`, `session:answers-revealed` emitido com distribuição e opção correta

**Revelar respostas (manual):**
- Dado host em sessão `QUESTION`
- Quando host emite `game:reveal-answers { pin }`
- Então timer cancelado, status → `REVEAL`, `session:answers-revealed` emitido para sala

**Placar parcial:**
- Dado sessão em `REVEAL`
- Quando host emite `game:show-leaderboard { pin }`
- Então status → `LEADERBOARD`, `session:leaderboard` emitido com ranking ordenado por score

**Encerramento:**
- Dado host emite `game:end { pin }` em qualquer estado ativo
- Então resultados persistidos no PostgreSQL, Redis limpo, `session:ended` emitido com ranking final

**Consulta de sessão (REST):**
- Dado qualquer cliente com PIN válido
- Quando `GET /api/sessions/:pin`
- Então `200` com estado atual da sessão (sem dados sensíveis de resposta)

## Edge Cases

1. **PIN collision**: gerar novo PIN se já existe no Redis (até 10 tentativas, depois lançar erro 503).
2. **Nickname duplicado**: emitir `session:error { code: "NICKNAME_TAKEN" }` ao socket tentador.
3. **Resposta fora do tempo**: ignorar silenciosamente (não emitir erro, apenas não computar).
4. **Resposta duplicada**: ignorar segunda resposta (Redis Hash `HSETNX` garante idempotência).
5. **Transição inválida**: emitir `session:error { code: "INVALID_TRANSITION", from, to }` ao socket que tentou.
6. **Host desconecta**: iniciar timer de 30s. Reconexão dentro do prazo restaura `hostSocketId` e cancela o timer.
7. **Sessão expirada no Redis (TTL)**: retornar `session:error { code: "SESSION_EXPIRED" }` em qualquer evento.
8. **game:start com quiz sem perguntas**: emitir `session:error { code: "QUIZ_EMPTY" }`.
9. **game:next-question sem mais perguntas**: emitir `session:error { code: "NO_MORE_QUESTIONS" }`.
10. **Jogador tenta entrar com sessão em QUESTION/REVEAL/FINISHED**: emitir `session:error { code: "SESSION_NOT_ACCEPTING" }`.
11. **Múltiplas conexões do mesmo host**: apenas a primeira é aceita como host; novas tentativas recebem `session:error { code: "HOST_ALREADY_CONNECTED" }`.
12. **Redis indisponível**: propagar erro com `session:error { code: "INTERNAL_ERROR" }`, logar, não quebrar o processo.

## Namespace Socket.io: `/game`

### Eventos Host → Server (apenas `hostSocketId`)

| Evento | Payload | Pré-condição |
|---|---|---|
| `game:start` | `{ pin }` | status = LOBBY |
| `game:next-question` | `{ pin }` | status = ACTIVE \| LEADERBOARD |
| `game:reveal-answers` | `{ pin }` | status = QUESTION |
| `game:show-leaderboard` | `{ pin }` | status = REVEAL |
| `game:end` | `{ pin }` | any active status |

### Eventos Aluno → Server

| Evento | Payload | Pré-condição |
|---|---|---|
| `player:join` | `{ pin, nickname, avatarId }` | status = LOBBY |
| `player:answer` | `{ pin, questionId, optionId, answeredInMs }` | status = QUESTION, dentro do timeLimitMs |

### Eventos Server → Clientes

| Evento | Payload | Destinatário |
|---|---|---|
| `session:player-joined` | `{ participant: ParticipantState, totalPlayers: number }` | Room `game:{pin}` |
| `session:question-start` | `{ question (sem isCorrect), questionIndex, totalQuestions, timeLimitSecs, questionStartedAt }` | Room `game:{pin}` |
| `session:timer-tick` | `{ remainingMs: number }` | Room `game:{pin}` |
| `session:answer-received` | `{ answeredCount: number, totalPlayers: number }` | Host socket only |
| `session:answers-revealed` | `{ correctOptionId, distribution: Record<string,number>, participants: ParticipantScore[] }` | Room `game:{pin}` |
| `session:leaderboard` | `{ ranking: ParticipantScore[], currentQuestionIndex, totalQuestions }` | Room `game:{pin}` |
| `session:ended` | `{ ranking: ParticipantScore[] }` | Room `game:{pin}` |
| `session:error` | `{ code: string, message: string }` | Socket que causou o erro |

> `session:timer-tick` deve ser emitido a cada segundo via `setInterval` enquanto status = QUESTION.
