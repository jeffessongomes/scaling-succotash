## Task 1 — AnswerRecord + Cache (RED → GREEN)

**Arquivos:** `game.types.ts`, `game-state.cache.ts`, `game-state.cache.test.ts`

### RED — Testes de cache (falharão até a implementação)

Atualizar `game-state.cache.test.ts`:
- `saveAnswer` deve serializar como `"opt-correct:5000"` no HSETNX
- `saveAnswer` deve aceitar 5 parâmetros (pin, questionId, participantId, optionId, answeredInMs)
- `saveAnswer` retorna `false` em duplicata
- `getAnswers` parseia `"opt-correct:5000"` → `{ optionId: 'opt-correct', answeredInMs: 5000 }`
- `getAnswers` retorna `{}` com hash vazio
- `getAnswers` parseia múltiplos participantes

### GREEN — Implementação

1. Adicionar `AnswerRecord` em `game.types.ts`
2. Atualizar `saveAnswer`: adicionar `optionId`, serializar `"${optionId}:${answeredInMs}"`
3. Atualizar `getAnswers`: retornar `Record<string, AnswerRecord>`, parsear via `indexOf(':')`

**Verificação:** `pnpm test apps/api/src/infrastructure/cache/__tests__/game-state.cache.test.ts`

## Task 2 — Socket (RED → GREEN)

**Arquivos:** `game.socket.ts`, `game.socket.test.ts`

### RED — Testes de socket (falharão até a implementação)

Atualizar `game.socket.test.ts`:
- Mock de `getAnswers` deve retornar `Record<string, AnswerRecord>`
- `player:answer` passa `optionId` para `saveAnswer`
- `revealAnswers` com resposta correta: `distribution['opt-correct'] === 1`, `participants[0].score > 0`
- `revealAnswers` com resposta errada: `distribution['opt-wrong'] === 1`, `distribution['opt-correct'] === 0`
- `revealAnswers` sem respostas: todos os `distribution` zerados

### GREEN — Implementação

1. `player:answer`: trocar `saveAnswer(pin, questionId, participant.id, answeredInMs)` por `saveAnswer(pin, questionId, participant.id, optionId, answeredInMs)` e remover `void optionId`
2. `revealAnswers`: atualizar loop para usar `answer.optionId` e `answer.answeredInMs`

**Verificação:** `pnpm test apps/api/src/features/game/__tests__/game.socket.test.ts`
